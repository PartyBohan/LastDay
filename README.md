# 视感 · Last Day · 离职交接

> 给视感所有同事的离职交接清单。15 分钟走完，不漏项。

**线上地址**：https://lastday.partykeys.org

---

## 一句话

视感同事决定离职后，从签字到走完最后一天，需要交接的事情**全部列在这里** — 一个页面、可勾选、自动保存进度、最后生成签字归档单。

---

## 为什么做这个

视感之前的离职流程靠 HR 一对一发文档、群里口头确认。结果常常出现：

- 离职同事不清楚自己该交什么 → 漏交，接班人接不住
- 设备 / 账号 / 密码散在各处 → 三个月后才发现某个广告账户没人管
- 财务 / 社保 / 年终 怎么算 → 离职后才反应过来要去问
- HR 重复回答相同问题 → 每个离职都跑一遍同样的对话

这个页面把**对接清单标准化**：每个人打开就知道下一步做什么、找谁、什么时候该完成。HR 同步参与，但不用再重复讲。

---

## 包含什么 (3 + 1 + 2 个模块)

**通用交接 · 3 模块**

1. **工作交接** — 项目 / SOP / 联系人 / 历史决策；在岗带新三步法 (看一遍 → 陪一遍 → 放一遍)；《工作交接表》流程
2. **设备 & 账号** — 笔记本 / 门禁 / SIM 卡归还清单；钉钉 / 飞书 / Git / Figma / 亚马逊 / Shopify / 广告 / 银行 后台权限处理；外部群 / 工作邮箱过渡
3. **财务 & 社保** — 末月工资节奏；五险一金停缴 / 转移；离职证明 / 个税 / 报销 / Exit Interview

**部门交接 · 1 模块** (按你部门自动展示)

- 产研中心 / 交付中心 / 国内电商 / 海外电商 — 每个部门的交接重点

**速查工具 · 2 模块**

- **离职清单** — 22 项可勾选 + 自动保存
- **找谁** — 操作问题对接路径表 (HR / 行政 / 财务 / 部门负责人)

---

## 谁用

- **正在办离职的同事** — 主用户，进首页填姓名 + 部门，按顺序走完
- **HR / 行政** — 同步参与，作为对接清单
- **直属上级 / 接班人** — 知道交接范围
- **管理者 / 合伙人** — 用管理者模式核对全部内容

---

## 怎么用

1. 浏览器打开 `lastday.partykeys.org`
2. 填**姓名 + 部门**，进入交接清单
3. 按顺序看完每个模块 — 看到了就算，没有测验、没有打分
4. 完成后生成**离职交接完成单**，可打印 / PDF 保存，本人 + HR 签字归档

进度自动存在浏览器里，下次打开继续。换浏览器 / 清缓存会重置。

---

## 设计原则

- **不评价、不催、不教育** — 进到这个页面前，离职已经谈完
- **直接给条目，不写"为什么这件事重要"** — 核心内容一项项排清楚就够
- **没有告别 / 校友 / 鸡汤内容** — 操作清单，不是情感系统
- **15 分钟以内走完** — 时间是同事的，少占用
